Version: 0.7.2

  1. NEW/Re-worked: This build total re-worked the Zone create process.
    Starting with this build (0.7.1+)High Availability/DisasterRecovery(HA/DR) has been build in from the start.
    What this means to you. if you create a new Zone in DC1 and DC2 is down the zone will continue to be processed in dc1, a failed job will be created/saved for latter to run.
    Once the issue has been resolved/fixed, the job can be re-submitted to continue.

    To help in the re-submit process, a new jobs list has been created, this will list all failed jobs (and running jobs) including a failed count, failed reason, etc.. to re-submit the job is as easy as clicking the re-submit option.
    Also, the zone process/percent completed in the detail section has been updated, it will now show you details per Data Center.
  2. Fixed: This version fixes an outstanding issue with SMF working in aSync mode only. we now added an extra check to each step in the process.
    This fix address many other issues, making sure things are executed in the proper order.
  3. Update: Added timing to file system refresh.
  4. Update: Update color on percent status bar.
  5. Update: Re-worked the zone expend strcuture, shuld fix the issue somtimes jumping while expanding.
